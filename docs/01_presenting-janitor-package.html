Introduction

The janitor package was created by Sam Firke

Its purpose was to make data nomalising as quick and swift as possible, since data cleaning is one of the most time consuming activities during data analysis and work.

This post will go over the janitor packages most characteristic functionalities.

A disclaimer: I had used the package lots of times before, specifically the clean_names() function. I will now try to learn of some other ones.

A messy data example

Data collected or found online is not always normalised, or say, clean. Data coming from excel files must be treated before it is useful for work with programs such as R, or Python.

Some indicators of messyness are:

  • strange column or variable names
  • empty rows or columns
  • consutant number columns
  • strange or different date formats

Here there’s an example of a messy data set found in the janitor github page.
It’s an excel file with data on fictional teachers HR information.
